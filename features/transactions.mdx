## Transaction types and statuses

Every transaction has a **type** and a **status**. Only `success` transactions count toward KPI totals.

**Types**

| Type | Description |
|---|---|
| `income` | Money received from a customer or other source |
| `expense` | Money spent on operations, consumables, or services |
| `refund` | A reversal of a previous income transaction |

**Statuses**

| Status | Meaning |
|---|---|
| `pending` | Recorded but not yet confirmed |
| `success` | Confirmed â€” included in KPI totals |
| `refunded` | Income that has been returned to the customer |
| `cancelled` | Voided â€” excluded from all calculations |

**Status workflow:** Transactions start as `pending` and move to `success` once confirmed, or to `cancelled` if voided. Income transactions can also move to `refunded`.

## Creating a transaction

<Steps>
  <Step title="Open the Add menu">
    Click the **Add** button in the top-right corner of the Transactions page. A dropdown shows three options.
  </Step>
  <Step title="Choose the transaction type">
    - **Record Payment** â€” creates an `income` transaction.
    - **Record Expense** â€” creates an `expense` transaction.
    - **Use Inventory** â€” consumes stock and automatically creates a `source: "inventory"` expense transaction.
  </Step>
  <Step title="Fill the form">
    Enter the description, amount, date, and optional customer or category. For inventory consumption, select the item and quantity.
  </Step>
  <Step title="Submit">
    Click **Add** (or **Log Usage** for inventory). The transaction appears in the table and the running balance updates immediately.
  </Step>
  <Step title="Confirm the transaction">
    New transactions default to `pending`. Use the Status dropdown in the table row to change it to `success` once payment or expense is confirmed.
  </Step>
</Steps>

## Filtering transactions

The toolbar above the table exposes the following filters. All filters stack â€” selecting multiple narrows the result set further.

| Filter | Options |
|---|---|
| Date range | Custom from / to date inputs with a **Clear dates** shortcut |
| Type | All, income, expense, refund |
| Status | All, success, pending, refunded, cancelled |
| Category | Derived from existing transaction categories on this site |
| Customer | Derived from customers linked to this site |

A full-text search box matches against `description`, `reference_no`, and `category`.

## Sources: manual vs inventory

Transactions with `source: "inventory"` are created automatically when you log inventory consumption from the Inventory page. They should not be edited directly â€” delete the consumption record from Inventory if a correction is needed.

## Export and import

**Export CSV** â€” downloads all rows currently visible (respecting active filters) with columns: Date, Reference, Description, Category, Customer, Type, Status, Qty, Unit Price, Total.

**Import CSV** â€” bulk-creates transactions from a spreadsheet. Required columns: `Type`, `Status`, `Quantity`, `Unit Price`, `Transaction Date`. Optional: `Description`, `Reference No`, `Category`.
